# HireNest CRM 2.0 - Technical Requirements Document (TRD)

## 1. Architectural Strategy
HireNest CRM runs alongside the underlying execution engine (HireNestOS). The relationship is strictly unidirectional in core execution updates, communicating via a system-wide Event Bus.

`[HireNest OS (AI, AST, Matches)]` ---> `[Event Bus]` ---> `[Sync Engine / API]` ---> `[HireNest CRM]`

## 2. Infrastructure & Datastore
**Platform:** Next.js / Vite React PWA + Supabase (PostgreSQL)

### 2.1 Core Collections
1. **`accounts`**: Enterprise entities. Replaces `clients`.
2. **`contacts`**: Actual people associated with `accounts`. 
3. **`requirements`**: Structured requests tied to `accounts` and mapping revenue. Replaces `jobs`.
4. **`vendors`**: Connected partners.
5. **`communications`**: Unified inbox data structure.
6. **`followups`**: Dedicated tasks spanning Requirements, Submissions, Revenue.
7. **`system_events`**: Immutable event ledger.
8. **`revenue_pipeline`**: Financial states indexed.

## 3. Communication Architecture
The `CommunicationCenter` integrates disjoint channels (Email, WhatsApp) under a single federated structure.
- **Data Model:** Threading based on `entity_id` and `entity_type` (Client, Vendor, Contact). 
- **System Events Integration:** Auto-inject system alerts directly into the communication thread timelines.

## 4. Revenue Architecture
Linear state machine tracking:
`Draft -> Sourced -> Submitted -> Interviewing -> Offered -> Placed -> Invoiced -> Paid`
Financial forecasts are calculated dynamically per account, driven by requirement statuses natively emitted from `system_events`.

## 5. Event Bus Interface
The fundamental engine component connecting OS and CRM:

```json
{
  "event_id": "uuid",
  "event_type": "INTERVIEW_SCHEDULED",
  "entity_type": "candidate",
  "entity_id": "123",
  "created_by": "system_os",
  "timestamp": "iso_string",
  "metadata": {}
}
```
CRM Subscriptions:
The `SyncEngine` inside the CRM listens to these states, updating the Timeline, Revenue Pipeline, and generating required `followups` implicitly without external human inputs.
